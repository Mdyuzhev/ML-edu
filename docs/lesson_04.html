Engineering

Сырые признаки редко подходят для модели напрямую. Учимся создавать новые фичи, кодировать категории числами и нормализовать данные — это часто важнее выбора алгоритма.

Примеры трансформаций признаков
country_code = "US" → country_US = 1, country_DE = 0 … get_dummies()
price_local = 118,300 KRW → price_usd = 84.0 fx_rates · map()
price_usd = 84.0 → price_scaled = 0.42 StandardScaler
has_discount = NaN / 30% → is_on_sale = 0 / 1 np.where()
🔢
Кодирование категорий
get_dummies() для country_code, gender, category — превращаем текст в числа
💱
Конвертация валют
Приводим все цены к USD через словарь курсов. Без этого сравнивать страны нельзя
⚖️
Нормализация
StandardScaler, MinMaxScaler — масштабируем признаки чтобы модель работала корректно
✨
Новые признаки
is_on_sale, price_segment (бюджет/средний/премиум), price_vs_median_country
