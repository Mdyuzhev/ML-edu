#02Что такое DataFrame

DataFrame — это главный объект библиотеки pandas. Думай о нём как об умной таблице: строки, колонки, индексы. Всё что можно делать в Excel — можно делать с DataFrame, только быстрее и на миллионах строк.

После строки df = pd.read_csv("Adidas_Global.csv", low_memory=False) переменная df содержит всю таблицу в памяти. df — просто имя переменной, принятое в data science сообществе.

Основные операции

Python Шпаргалка по DataFrame 
# Размер таблицы
df.shape          # (44888, 35) — кортеж (строки, колонки)
df.shape[0]       # 44888 — только строки
len(df)           # 44888 — тоже количество строк

df.head()         # первые 5 строк
df.tail()         # последние 5 строк
df.sample(5)      # 5 случайных строк

# Выбор данных
df['price_local']                    # одна колонка → Series
df[['product_name', 'price_local']]  # несколько → DataFrame

# Фильтрация строк
df[df['country_code'] == 'US']           # только США
df[df['discount_pct'] > 0]             # товары со скидкой
df[(df['country_code'] == 'US') & (df['price_local'] > 100)]

# Главное правило: df не меняется без явного присваивания
df_us = df[df['country_code'] == 'US'].copy()

#04Первичная статистика — describe()

• rating_count: min = -99 — технический маркер «нет данных» вместо NaN. Типичный «мусор» в реальных данных.

Фильтрация и сортировка Series:

• df.isnull().sum() / len(df) * 100 — считает процент пропусков по каждой колонке

• missing[missing > 1] — фильтрация: оставить только колонки с пропусками больше 1%. Квадратные скобки с условием — один из главных приёмов pandas

• Порог 1% — практическое правило: меньше 1% пропусков не влияют на модель

• .sort_values(ascending=False) — сортировка по убыванию. True — наоборот

In [8] Визуализация колонок с пропусками > 31% 
import matplotlib.pyplot as plt

missing_critical = missing[missing > 31].sort_values(ascending=False)

plt.figure(figsize=(8, 8))
plt.pie(missing_critical, labels=missing_critical.index, autopct='%1.1f%%', startangle=140)
plt.title('Колонки с пропусками > 31%')

Правило выбора стратегии: пропусков >70% → удалить колонку. Пропуски имеют смысл (нет скидки, нет отзывов) → заполнить 0. Пропуски случайные → заполнить медианой или модой.

#07Итоги урока

Что мы узнали о датасете

  • 44,888 строк = SKU (один размер одного товара), 1,990 уникальных моделей по 10 странам
  • Аномалии: rating_count = -99 как замена NaN, отрицательные значения discount_pct
  • color_name (90%) и badge_texts (75%) — кандидаты на удаление
  • brand_name и in_stock — константы, бесполезны как признаки
  • price_local и sale_price_local = идеальная корреляция (мультиколлинеарность)
  • category на 6 языках (53 уникальных значения) — требует нормализации
  • Китай — selection bias: всего 904 строки (2%), закрытая экосистема
